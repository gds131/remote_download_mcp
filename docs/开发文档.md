# 远程下载管理MCP服务开发文档

## 1. 项目概述

基于MCP协议的远程下载服务，支持用户通过外挂配置文件自定义多个下载器实例（如qb1、qb2、tr1等），部署于Docker。

## 2. 项目结构

```
remote-download-mcp/
├── src/
│   ├── __init__.py
│   ├── server.py              # MCP Server主入口
│   ├── config.py              # 配置加载与管理
│   └── downloaders/
│       ├── __init__.py
│       ├── base.py            # 下载器抽象基类
│       ├── qbittorrent.py     # qBittorrent实现
│       └── transmission.py    # Transmission实现
├── config/

├── docker/
│   ├── Dockerfile

├── requirements.txt
└── README.md
```

## 4. 核心代码实现

### 4.1 配置管理模块 (src/config.py)

```python
import yaml
from pathlib import Path
from typing import Dict, Any

CONFIG_PATH = Path("/app/config/downloaders.yaml")

class DownloaderConfig:
    def __init__(self, alias: str, type_: str, url: str, username: str, password: str):
        self.alias = alias
        self.type = type_
        self.url = url
        self.username = username
        self.password = password

class ConfigManager:
    _instance = None
    _config: Dict[str, DownloaderConfig] = {}
    _default_alias: str = None
    
    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._load_config()
        return cls._instance
    
    def _load_config(self):
        """从YAML文件加载配置"""
        if not CONFIG_PATH.exists():
            raise FileNotFoundError(f"配置文件不存在: {CONFIG_PATH}")
        
        with open(CONFIG_PATH, 'r', encoding='utf-8') as f:
            data = yaml.safe_load(f)
        
        for alias, cfg in data.get('downloaders', {}).items():
            self._config[alias] = DownloaderConfig(
                alias=alias,
                type_=cfg['type'],
                url=cfg['url'],
                username=cfg['username'],
                password=cfg['password']
            )
        
        self._default_alias = data.get('default_downloader')
    
    def get_downloader(self, alias: str) -> DownloaderConfig:
        if alias not in self._config:
            raise ValueError(f"未知的下载器别名: {alias}，可用别名: {list(self._config.keys())}")
        return self._config[alias]
    
    def list_aliases(self) -> list[str]:
        return list(self._config.keys())
    
    def get_default(self) -> str:
        return self._default_alias

settings = ConfigManager()
```

### 4.2 下载器抽象基类 (src/downloaders/base.py)

```python
from abc import ABC, abstractmethod

class DownloaderBase(ABC):
    """下载器抽象基类"""
    
    def __init__(self, url: str, username: str, password: str):
        self.url = url
        self.username = username
        self.password = password
    
    @abstractmethod
    async def add_torrent(self, url: str, save_path: str) -> dict:
        """添加下载任务"""
        pass
    
    @abstractmethod
    async def ping(self) -> bool:
        """健康检查"""
        pass
```

### 4.3 qBittorrent实现 (src/downloaders/qbittorrent.py)

```python
from .base import DownloaderBase
import qbittorrentapi

class QBittorrentClient(DownloaderBase):
    """qBittorrent下载器客户端"""
    
    def __init__(self, url: str, username: str, password: str):
        super().__init__(url, username, password)
        self.client = None
    
    async def _ensure_connected(self):
        """确保连接到qBittorrent"""
        if self.client is None:
            self.client = qbittorrentapi.Client(
                host=self.url,
                username=self.username,
                password=self.password
            )
            try:
                self.client.auth_log_in()
            except Exception as e:
                raise ConnectionError(f"无法连接到qBittorrent: {str(e)}")
    
    async def add_torrent(self, url: str, save_path: str) -> dict:
        """添加下载任务"""
        await self._ensure_connected()
        try:
            result = self.client.torrents_add(
                urls=url,
                save_path=save_path
            )
            return {"success": True, "info": result}
        except Exception as e:
            raise Exception(f"添加下载任务失败: {str(e)}")
    
    async def ping(self) -> bool:
        """健康检查"""
        try:
            await self._ensure_connected()
            return True
        except:
            return False
```

### 4.4 Transmission实现 (src/downloaders/transmission.py)

```python
from .base import DownloaderBase
from transmission_rpc import Client as TransmissionClientRPC

class TransmissionClient(DownloaderBase):
    """Transmission下载器客户端"""
    
    def __init__(self, url: str, username: str, password: str):
        super().__init__(url, username, password)
        self.client = None
    
    async def _ensure_connected(self):
        """确保连接到Transmission"""
        if self.client is None:
            try:
                self.client = TransmissionClientRPC(
                    url=self.url,
                    username=self.username,
                    password=self.password
                )
            except Exception as e:
                raise ConnectionError(f"无法连接到Transmission: {str(e)}")
    
    async def add_torrent(self, url: str, save_path: str = None) -> dict:
        """添加下载任务"""
        logger.info(f"添加下载任务到Transmission: {self.url}, 保存路径: {save_path}")
        await self._ensure_connected()
        try:
            params = {"torrent": url}
            if save_path is not None:
                params["download_dir"] = save_path
            result = self.client.add_torrent(**params)
            logger.info(f"下载任务添加成功: {url}")
            return {"success": True, "info_hash": result.hashString}
        except Exception as e:
            logger.error(f"添加下载任务失败: {str(e)}")
            raise Exception(f"添加下载任务失败: {str(e)}")
    
    async def ping(self) -> bool:
        """健康检查"""
        try:
            await self._ensure_connected()
            return True
        except:
            return False
```

### 4.5 MCP服务主入口 (src/server.py)

```python
import logging
import os
from mcp.server.fastmcp import FastMCP
from src.config import settings
from src.downloaders.qbittorrent import QBittorrentClient
from src.downloaders.transmission import TransmissionClient

# 配置日志
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    handlers=[
        logging.StreamHandler()
    ]
)

logger = logging.getLogger(__name__)

mcp = FastMCP("remote-download")

# 下载器实例缓存
_downloader_instances: dict[str, any] = {}

def get_downloader(alias: str):
    """根据别名获取或创建下载器实例"""
    if alias not in _downloader_instances:
        cfg = settings.get_downloader(alias)
        
        if cfg.type == "qbittorrent":
            _downloader_instances[alias] = QBittorrentClient(
                url=cfg.url,
                username=cfg.username,
                password=cfg.password
            )
        elif cfg.type == "transmission":
            _downloader_instances[alias] = TransmissionClient(
                url=cfg.url,
                username=cfg.username,
                password=cfg.password
            )
        else:
            raise ValueError(f"不支持的下载器类型: {cfg.type}")
    
    return _downloader_instances[alias]

@mcp.tool()
async def add_download(
    download_url: str,
    downloader_alias: str,
    save_path: str = None
) -> dict:
    """提交下载任务到指定远端下载器"""
    downloader = get_downloader(downloader_alias)
    result = await downloader.add_torrent(
        url=download_url,
        save_path=save_path
    )
    return {
        "success": True,
        "message": "下载任务已提交",
        "downloader": downloader_alias,
        "save_path": save_path
    }

@mcp.tool()
async def list_downloaders() -> dict:
    """列出所有配置的下载器别名"""
    return {
        "aliases": settings.list_aliases(),
        "default": settings.get_default(),
        "count": len(settings.list_aliases())
    }

@mcp.get("/health")
async def health_check():
    """健康检查"""
    status = {}
    for alias in settings.list_aliases():
        try:
            dl = get_downloader(alias)
            await dl.ping()
            status[alias] = "connected"
        except Exception as e:
            status[alias] = f"disconnected: {str(e)}"
    
    return {
        "status": "ok" if any(s == "connected" for s in status.values()) else "degraded",
        "downloaders": status
    }

if __name__ == "__main__":
    mcp.run(transport="streamable-http", port=8000)
```

```yaml
downloaders:
  # qBittorrent 实例1
  qb1:

  qb2:

    username: test
    password: testadmin
    
  # Transmission 实例1
  tr1:
    type: transmission
    url: http://192.168.10.222:8080
    username: test
    password: testadmin

# 可选：默认下载器
default_downloader: qb1
```

### 5.2 配置字段说明

| 字段                   | 说明                | 必填 | 可选值                           |
| -------------------- | ----------------- | -- | ----------------------------- |
| `downloaders`        | 下载器配置字典           | 是  | -                             |
| `<alias>`            | 用户自定义别名（如qb1/tr2） | 是  | 任意字符串                         |
| `type`               | 下载器类型             | 是  | `qbittorrent`, `transmission` |
| `url`                | Web UI/RPC地址      | 是  | 完整URL                         |
| `username`           | 用户名               | 是  | -                             |
| `password`           | 密码                | 是  | -                             |
| `default_downloader` | 默认下载器别名           | 否  | 必须是存在的alias                   |

## 6. Docker配置

### 6.1 网络配置说明

MCP服务的网络配置采用环境变量方式，支持用户自定义容器IP和网络网段，以避免与下载器IP不在同一网段导致的请求失败。具体工作原理如下：

1. **Docker网络配置**：在`docker-compose.yml`中，定义了自定义网络`mcp-network`，支持通过环境变量配置网段和网关
2. **容器IP配置**：通过`MCP_CONTAINER_IP`环境变量指定容器的固定IP地址
3. **端口配置**：使用`${MCP_PORT:-8000}:${MCP_PORT:-8000}`格式，确保容器内外端口一致
4. **环境变量传递**：通过环境变量将网络配置传递给容器内部

**网络配置环境变量**：

- `MCP_SUBNET`：网络网段，默认`172.20.0.0/16`
- `MCP_GATEWAY`：网络网关，默认`172.20.0.1`
- `MCP_CONTAINER_IP`：容器固定IP，默认`172.20.0.2`
- `MCP_PORT`：服务端口，默认`8000`

这种配置方式确保了：

- 当用户在启动容器时指定网络配置，服务会自动使用该配置
- 未指定时，使用默认网络配置
- 容器IP与下载器IP可以配置在同一网段，避免网络访问问题
- 容器内外端口保持一致，避免端口映射错误

### 6.2 Dockerfile

```dockerfile
FROM python:3.10-slim

WORKDIR /app

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制源码
COPY src/ ./src/

# 创建配置目录（外挂卷挂载点）
RUN mkdir -p /app/config

EXPOSE 8000

HEALTHCHECK --interval=30s --timeout=10s --retries=3 \
    CMD python -c "import urllib.request, os; port = os.getenv('MCP_PORT', '8000'); urllib.request.urlopen('http://localhost:' + port + '/health')"

CMD ["python", "-m", "src.server"]
```

### 6.3 docker-compose.yml

```yaml
version: '3.8'

networks:
  mcp-network:
    driver: bridge
    ipam:
      config:
        - subnet: ${MCP_SUBNET:-172.20.0.0/16}
          gateway: ${MCP_GATEWAY:-172.20.0.1}

services:
  mcp-download:
    build:
      context: .
      dockerfile: docker/Dockerfile
    image: remote-download-mcp:latest
    container_name: download-mcp
    ports:
      - "${MCP_PORT:-8000}:${MCP_PORT:-8000}"
    volumes:
      # 外挂配置文件
      - ./config:/app/config:ro
    environment:
      - MCP_HOST=0.0.0.0
      - MCP_PORT=${MCP_PORT:-8000}
      - MCP_CONTAINER_IP=${MCP_CONTAINER_IP:-172.20.0.2}
    networks:
      mcp-network:
        ipv4_address: ${MCP_CONTAINER_IP:-172.20.0.2}
    restart: unless-stopped
```

## 8. MCP工具使用说明

### 8.1 add\_download

**功能**: 提交资源到指定远端下载器进行下载

**参数**:

- `download_url`: 资源下载地址(magnet/http/ftp)，必填
- `save_path`: 保存路径(相对于下载器根目录)，默认"/downloads"
- `downloader_alias`: 下载器别名，对应配置文件中的key（如qb1/tr1），必填

**返回值**:

```json
{
  "success": true,
  "message": "下载任务已提交",
  "downloader": "qb1",
  "save_path": "/downloads"
}
```

### 8.2 list\_downloaders

**功能**: 列出当前配置的所有可用下载器别名

**参数**: 无

**返回值**:

```json
{
  "aliases": ["qb1", "tr1"],
  "default": "qb1",
  "count": 2
}
```

## 9. 常见问题处理

### 9.1 配置文件不存在

- 错误信息: `FileNotFoundError: 配置文件不存在: /app/config/downloaders.yaml`
- 解决方法: 确保挂载了配置文件，或在config目录中创建downloaders.yaml文件

### 9.2 下载器连接失败

- 错误信息: `ConnectionError: 无法连接到qBittorrent: ...`
- 解决方法: 检查下载器URL、用户名和密码是否正确，确保下载器服务正常运行

### 9.3 不支持的下载器类型

- 错误信息: `ValueError: 不支持的下载器类型: ...`
- 解决方法: 确保配置文件中的type字段值为"qbittorrent"或"transmission"

## 10. 扩展指南

### 10.1 添加新的下载器类型

1. 在`src/downloaders/`目录下创建新的下载器实现文件
2. 继承`DownloaderBase`抽象基类
3. 实现`add_torrent`和`ping`方法
4. 在`server.py`的`get_downloader`函数中添加新的类型判断

### 10.2 配置热加载

当前实现不支持配置热加载，如需实现此功能，可以:

1. 在`ConfigManager`中添加文件监控
2. 当配置文件变化时重新加载配置
3. 清空下载器实例缓存，使新配置生效

## 11. 安全注意事项

1. 配置文件中包含敏感信息（用户名和密码），请确保配置文件权限设置正确
2. 建议在生产环境中使用HTTPS保护API通信
3. 限制MCP服务的访问范围，仅允许受信任的客户端访问
4. 定期更新依赖库，修复安全漏洞

## 12. 性能优化

1. 下载器实例缓存: 避免重复创建客户端连接
2. 异步操作: 使用async/await提高并发处理能力
3. 错误处理: 合理的异常捕获和处理，提高服务稳定性
4. 健康检查: 定期检查下载器状态，及时发现问题
